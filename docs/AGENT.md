# Guto CLI - Agent Rules & Standards

These rules must be followed by all AI agents (Gemini, Claude, etc.) working on this repository.

1. **Preserve Consolidation:** Do not change or refactor logic, configurations, or established patterns that are already consolidated unless specifically instructed to do so.
2. **Language:** Always communicate, write code, and document in **English**.
3. **Intent-Based Commenting:** Comments should explain **WHY** a piece of logic exists (the intent or business rationale), not **WHAT** the code is doing. Implementation details should be clear from the code itself.
4. **Documentation Sync:** Every feature change, bug fix, or configuration update MUST be reflected in the documentation (`README.md`, `GEMINI.md`, etc.) immediately.
5. **Self-Contained Semantic Commits:** Commits must be self-contained (atomic), written in **English**, and follow **Conventional Commits** (Semantic Commits) standards (e.g., `feat:`, `fix:`, `docs:`, `refactor:`).
